GitHub / HW_2

1. На локальном репозитории сделать ветки для:
- Postman --> git branch Postman 
- Jmeter --> git branch Jmeter 
- CheckLists --> git branch CheckLists 
- Bug Reports --> git branch BugReports 
- SQL --> git branch SQL 
- Charles --> git branch Charles
- Mobile testing --> git branch MobileTesting

2. Запушить все ветки на внешний репозиторий - git push -u origin name_of_the_branch

3. В ветке Bug Reports сделать текстовый документ со структурой баг репорта --> git checkout BugReport --> touch bug_report.txt --> vim bug_report.txt

4. Запушить структуру багрепорта на внешний репозиторий --> git add bug_report.txt --> git commit -a "Created file bug_report.txt" --> git push

5. Вмержить ветку Bug Reports в Main --> git checkout main --> git merge BugReports

6. Запушить main на внешний репозиторий --> git push

7. В ветке CheckLists набросать структуру чек листа --> git checkout CheckLists --> touch check_list.txt --> vim check_list.txt

8. Запушить структуру на внешний репозиторий --> git add check_list.txt --> git commit -a "Created check_list.txt --> git push

9. На внешнем репозитории сделать Pull Request ветки CheckLists в main --> Go to CheckLists --> Compare and pull request --> Create pull request --> Merge pull request --> Confirm merge
10. Синхронизировать Внешнюю и Локальную ветки Main --> git pull
